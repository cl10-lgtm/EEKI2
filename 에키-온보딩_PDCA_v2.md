## P — Plan (기획 목표 및 설계 원칙)

### 기능 목표

1. **최민준 경로(MVP)**: 앱 실행 후 30초 이내 첫 경로 결과 제공
2. **박서연 경로(v1.1)**: 교통카드 선택 → 충전 안내 → 잔액 알림 → 오프라인 지도 다운로드까지 단일 플로우로 완결
3. **공통**: 온보딩 이탈 후 재진입 시 이전 단계 유지, 완료 전에도 핵심 기능 사용 가능

### 설계 원칙

- **선택지 최대 3개** — 정보 과다가 핵심 페인포인트 (U6: "수많은 안내로 헷갈림")
- **강제 입력 없음** — 모든 단계 스킵 가능, 설정 전에도 경로 검색 작동
- **MVP = 최민준 경로** — 즉시 실행형 유저가 입소문 만든다
- **오프라인 우선 설계** — 지하 LTE 불안정 환경 기본 가정

### FEAT-02. 즉시 경로 검색 ⭐ MVP

**트리거**: FEAT-01에서 "즉시 실행" 선택 또는 5초 무응답

**목적**: 준비 없이 현장에서 앱을 켠 최민준이 30초 안에 탑승 정보를 얻게 함
> U9: *"구글 맵에 의존하면서도 실제 진행 방향이 맞는지 확인할 수 없어 막막함"*
> 최민준: *"그냥 켜면 바로 알려주는 앱 하나 있었으면"*

**API 명세**

```
GET /api/v1/route
Request:
  {
    "origin_station_id": "string",   // GPS 감지 or 수동 입력
    "dest_query": "string",          // 자유 텍스트 검색
    "card_type": "suica|pasmo|travel_wallet|cash|null"
  }

Response:
  {
    "routes": [
      {
        "duration_min": 8,
        "fare_jpy": 200,
        "steps": [
          {
            "type": "subway",
            "line": "福岡市地下鉄空港線",
            "line_ko": "후쿠오카 공항선",
            "board_platform": "1번 플랫폼",
            "direction": "메이노하마 방면",
            "stops": 2,
            "alight_station_ko": "텐진",
            "exit_number": "7번 출구",
            "exit_destination": "텐진 지하상가 방면"
          }
        ]
      }
    ]
  }
```

**화면 — 검색 입력**
```
┌──────────────────────────────────────┐
│  지금 어디서 어디로?                  │
│                                      │
│  출발  [ 하카타역 🔄 ]  ← GPS 자동감지│
│  도착  [ 텍스트 입력... ]             │
│                                      │
│       [ 경로 찾기 ]                   │
└──────────────────────────────────────┘
```

**화면 — 결과**
```
┌──────────────────────────────────────┐
│  하카타 → 텐진   약 8분   ¥200       │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  🚇  후쿠오카 공항선 탑승            │
│      1번 플랫폼 · 메이노하마 방면     │
│      2정거장 후 텐진에서 하차         │
│                                      │
│  🚪  7번 출구로 나가세요             │
│      텐진 지하상가 방면              │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  💳 교통카드 설정하면 잔액 알림 가능  │
│  [ 나중에 ]          [ 지금 설정 ]   │
└──────────────────────────────────────┘
```

**GPS 자동 감지 로직**
```
1. GPS 좌표 → 반경 300m 내 역 목록 조회
2. 복수 역 감지 시 → "어느 역에 계신가요?" 선택 UI
3. GPS 실패 or 정확도 < 70% → 수동 입력 모드 전환
4. 오프라인 → 로컬 캐시 역 목록으로 자동완성만 제공,
             경로 계산 불가 안내 + 다운로드 유도
```

**예외 처리**
| 케이스 | 처리 방식 |
|--------|----------|
| 경로 없음 | "직접 연결 노선 없음 — 환승 경로 표시" |
| 목적지 인식 불가 | 자동완성 후보 3개 제시 |
| 운행 종료 시간 | "막차 지남 — 내일 첫차 정보 표시" |
| API 타임아웃 (> 5s) | 로컬 캐시 경로 제공 + "실시간 정보 아님" 배너 |

### FEAT-03. 교통카드 온보딩 (v1.1)

**트리거**: FEAT-01 "사전 준비" 선택 또는 FEAT-02 결과 화면 하단 "지금 설정" 탭

**목적**: IC카드 구매 → 충전 → 잔액 확인이 파편화된 문제를 앱 내 원스톱으로 해결
> U9: *"구매처와 방법에 대한 설명 부족으로 인한 시간 요소"*
> U6: *"교통카드 잔액 표시가 안 되어서 그냥 찍었다가 잔액부족으로 뜸"*

**카드 데이터 스키마**
```json
{
  "card_id": "suica",
  "name_ko": "Suica",
  "coverage": ["도쿄", "오사카", "후쿠오카", "전국 대부분"],
  "purchase_locations": [
    { "type": "ticket_machine", "desc_ko": "역내 초록색 자동발매기" },
    { "type": "convenience",    "desc_ko": "세븐일레븐 (일부 점포)" }
  ],
  "min_deposit_jpy": 500,
  "recommended_initial_jpy": 3000,
  "atm_keyword_ja": "チャージ",
  "unsupported_lines": [],
  "nfc_readable": true
}
```

**Step 1 — 카드 선택**
```
┌──────────────────────────────────────┐
│  어떤 교통카드를 쓸 예정인가요?       │
│                                      │
│  [ Suica        ] 전국 거의 모든 노선 │
│  [ Pasmo        ] 도쿄·간토 중심      │
│  [ 트래블월렛   ] 한국 카드로 충전    │
│                                      │
│  아직 모르겠어요 →                    │
└──────────────────────────────────────┘
```

**Step 2 — 맞춤 안내** (카드별 분기)
```
Suica 선택 시:
  - 구매: 역내 초록색 자동발매기 / 세븐일레븐
  - 초기 충전 권장: ¥3,000
  - 주의: 일부 사철 노선 사용 불가
  - 충전 ATM 찾는 법: 기계에서 "チャージ" 버튼

트래블월렛 선택 시:
  - 구매 불필요 (앱 내 발급)
  - 한국 카드로 엔화 충전
  - 주의: 일부 역 전용 노선 불가
  - U4: "빠진 요금 일일이 확인 불편" → 에키 잔액 알림 연동 안내
```

**Step 3 — 잔액 알림 설정**

```
Request (로컬 저장):
  {
    "card_type": "suica",
    "alert_threshold_jpy": 1000,
    "nfc_sync_enabled": true
  }

NFC 잔액 읽기 (iOS / Android):
  iOS  → CoreNFC (iOS 13+), NDEF 태그 읽기
  Android → NFC Adapter, Felica 태그 읽기
  실패 시 → 수동 잔액 입력 모드 폴백
```

**예외 처리**
| 케이스 | 처리 방식 |
|--------|----------|
| NFC 미지원 기기 | 수동 입력 + "충전 전 잔액 확인" 리마인더 |
| 카드 미선택 스킵 | 기본값 미설정, 홈에서 설정 배너 노출 |
| Step 중간 이탈 | 진행 단계 로컬 저장, 재진입 시 해당 Step부터 재개 |

### FEAT-05. 온보딩 완료 상태 관리

**목적**: 중간 이탈 유저 복귀율 확보, 완료 전에도 핵심 기능 사용 가능

**상태 저장 구조**
```json
{
  "onboarding": {
    "completed_at": null,
    "path": "instant | prepare",
    "steps": {
      "branch_selected": true,
      "first_route_searched": true,
      "card_type": "suica",
      "card_setup_done": false,
      "map_downloaded": false,
      "tutorial_done": false
    },
    "nudge_dismissed": false
  }
}
```

**홈 화면 배너 로직**
```
IF steps.card_setup_done == false AND nudge_dismissed == false:
  → "💳 교통카드 설정하면 잔액 알림 받을 수 있어요" 배너 노출
  → 닫기(X) 클릭 시 nudge_dismissed = true

IF steps.map_downloaded == false AND steps.card_setup_done == true:
  → "📍 오프라인 지도 저장하면 지하에서도 길 찾아요" 배너 노출

IF 모든 steps == true:
  → 배너 소멸, completed_at 기록
```

## C — Check (검증 지표)

### 정량 지표

| 지표 | 목표값 | 측정 이벤트 |
|------|--------|------------|

| 즉시 실행 → 첫 결과 이탈 없이 완료 | ≥ 75% | `route_result_shown` 후 5초 이상 체류 |

| NFC 잔액 연동 성공률 | ≥ 60% (NFC 지원 기기 한정) | `nfc_read_success` / `nfc_attempted` |

## A — Act (개선 트리거 및 로드맵)

### 즉시 개선 트리거

| 상황 | 기준 | 액션 |
|------|------|------|
| 첫 경로 검색 30초 초과 | 중앙값 > 45초 | GPS 감지 타임아웃 단축 (3s → 1.5s), 수동 입력 기본 노출 검토 |
| 즉시 실행 이탈 | 결과 화면 이탈 > 40% | 결과 화면 정보 과다 여부 점검, 출구 정보 fold 처리 |
| 카드 설정 완료율 저조 | < 30% | Step 수 3→2 축소, "나중에" 이후 재유도 타이밍 조정 |
| 오프라인 지도 다운 저조 | < 15% | 다운로드 혜택 메시지 강화 또는 첫 경로 검색 후 자동 추천 |
| NFC 연동 실패 급증 | 실패율 > 50% | 수동 입력 모드 기본값으로 전환, NFC 안내 개선 |
