## 퍼소나별 핵심 시나리오

| 구분 | 박서연 (P1) | 최민준 (P2) |
|------|------------|------------|
| 진입 시점 | 여행 출발 3일 전, 집에서 앱 설치 | 하카타 역 도착 직후, 개찰구 앞에서 실행 |
| 상태 | 준비하고 싶지만 어디서 시작해야 할지 모름 | 준비 없음, 지금 당장 어디 가야 할지만 알고 싶음 |
| 성공 기준 | "출발 전에 이미 다 세팅됐다"는 안도감 | "켜자마자 바로 탈 수 있다"는 확신 |

## P — Plan (기획 목표 및 설계 원칙)

### 온보딩 목표

1. 첫 실행 60초 안에 유저가 핵심 가치를 경험하게 한다
2. 박서연 경로(사전 준비형)와 최민준 경로(즉시 실행형) 두 가지를 분기 지원한다
3. 설정 완료 전에 이탈하더라도 최소한 경로 검색은 가능하게 한다

### 설계 원칙

- **선택지는 최대 3개** — 정보 과다가 핵심 페인포인트이므로 온보딩도 동일 원칙 적용
- **강제 입력 없음** — 모든 설정은 "나중에 하기" 탈출구 제공
- **진행률 표시** — 얼마나 남았는지 항상 보여줘 이탈 방지

## D — Do (기능 명세)

---

### FEAT-01. 진입 분기 선택

**트리거**: 앱 최초 실행 (또는 로그인 완료 직후)

**화면**: 단일 선택 카드 2종

| 옵션 | 라벨 | 연결 플로우 |
|------|------|------------|
| A | "여행 전에 미리 준비할게요" | FEAT-02 → FEAT-03 → FEAT-04 순서 진행 |
| B | "지금 당장 탈 곳 찾고 싶어요" | 즉시 경로 검색 화면으로 이동 (FEAT-05) |

**예외 처리**
- 5초 무응답 시 옵션 B(즉시 실행)로 자동 진입
- 최초 실행이 아닌 경우 이 화면 스킵

---

### FEAT-02. 교통카드 온보딩 (사전 준비 경로)

**목적**: U9 *"구매처와 방법에 대한 설명 부족으로 인한 시간 요소"* 해결

**단계별 명세**

```
Step 1 — 카드 선택 (1/3)
  ┌─────────────────────────────┐
  │  어떤 카드를 쓸 예정인가요?  │
  │                             │
  │  [ Suica        ]           │
  │  [ Pasmo        ]           │
  │  [ 트래블월렛   ]           │
  │                             │
  │  아직 모르겠어요 →          │
  └─────────────────────────────┘

Step 2 — 구매 방법 안내 (2/3)
  선택한 카드에 따라 맞춤 안내 표시
  - 구매 가능 장소 (역명 + 기기 위치)
  - 초기 충전 권장 금액
  - 주의사항 (일부 노선 사용 불가 등)

Step 3 — 잔액 알림 설정 (3/3)
  ┌─────────────────────────────┐
  │  잔액이 얼마 이하일 때      │
  │  알림을 드릴까요?           │
  │                             │
  │  [ 500엔 ] [ 1,000엔 ] [ 직접 입력 ] │
  └─────────────────────────────┘
```

**출력 결과**: 카드 종류, 알림 임계값 로컬 저장

---

### FEAT-03. 주요 역 오프라인 지도 다운로드

**목적**: U8 *"신주쿠 역같은 경우 다수의 출구로 인해 위치 찾기가 어려움"* / 지하 LTE 불안정 대비

**명세**

- 목적지 도시 선택 시 해당 복잡 역 자동 추천 (예: 도쿄 선택 → 신주쿠·시부야·도쿄역 추천)
- 각 역 지도 용량 표시 (예: "신주쿠역 · 2.3MB")
- 일괄 다운로드 / 개별 선택 모두 지원
- 다운로드 완료 시 "지하에서도 작동해요" 토스트 메시지

**포함 데이터**
- 출구 번호 + 출구별 인접 시설 (편의점, 화장실, 코인락커)
- 환승 홈 위치
- 안내소 위치

---

### FEAT-04. 역무원 대화 카드 소개

**목적**: U6 *"오래 번역 시에 눈치 보이고, 줄에 대한 부담감"* 해결

**온보딩 내 소개 방식**
- 30초 이내 완료되는 인터랙티브 튜토리얼 1회 제공
- "이런 상황에 써보세요" 예시 3가지 보여주기
  1. 잔액 부족으로 개찰구 막혔을 때
  2. 열차를 잘못 탔을 때
  3. 짐 보관함 위치를 물어볼 때
- 실제 카드 화면 미리보기 제공

**카드 데이터 구조**
```
{
  situation: "잔액 부족",
  korean_desc: "교통카드 잔액이 부족해요",
  japanese_display: "ICカードの残高が不足しています",
  furigana: "アイシーカードのざんだかがふそくしています"
}
```

---

### FEAT-05. 즉시 실행 경로 검색 (최민준 경로)

**목적**: 준비 없이 현장에서 실행한 유저가 30초 안에 첫 경로를 얻게 함

**명세**

```
진입 즉시 보이는 화면:
  ┌─────────────────────────────┐
  │  지금 어디서 어디로?        │
  │                             │
  │  현재 위치 [자동 감지 중..] │
  │  목적지   [________________]│
  │                             │
  │  [ 경로 찾기 ]              │
  └─────────────────────────────┘
```

- GPS 자동 감지로 현재 역명 자동 입력 (실패 시 수동 입력)
- 결과 화면에서 **탑승 플랫폼 번호 + 출구 번호** 함께 표시
- 경로 확인 후 "교통카드 설정도 해볼까요?" 넛지 배너 노출 (닫기 가능)

---

### FEAT-06. 온보딩 완료 상태 저장 및 재진입

**명세**

- 온보딩 각 단계 완료 여부 로컬 저장
- 중간 이탈 후 재진입 시 완료된 단계는 스킵, 미완료 단계부터 재개
- 홈 화면 상단에 "설정 완료하기" 배너 — 온보딩 완전 완료 시 자동 소멸
- 설정 메뉴에서 언제든 재설정 가능

## C — Check (검증 지표)

### 정량 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| 온보딩 완료율 | ≥ 60% | FEAT-01 진입 → FEAT-04 완료 |
| 즉시 실행 선택 비율 | 측정 후 분기 최적화 | FEAT-01 옵션 B 선택 수 |
| 카드 설정 완료율 | ≥ 45% | FEAT-02 Step 3 완료 수 |
| 오프라인 지도 다운로드율 | ≥ 30% | FEAT-03 다운로드 완료 수 |
| 첫 경로 검색까지 소요 시간 | ≤ 60초 | 앱 실행 → 첫 검색 결과 확인 |

### 정성 지표 (출시 후 2주 내 인터뷰)

- "이 앱 쓰면 뭔가 다르다"는 느낌을 받았는가?
- 온보딩 중 멈추거나 헷갈린 지점이 있었는가?
- 역무원 카드를 실제로 사용했는가?

## A — Act (개선 트리거 및 다음 단계)

### 즉시 개선 트리거

| 상황 | 기준 | 액션 |
|------|------|------|
| 온보딩 완료율 저조 | < 40% | FEAT-02 Step 수 축소 (3→2단계) 검토 |
| 즉시 실행 선택 압도적 | > 80% | 사전 준비 경로를 설정 메뉴로 이동, 온보딩 단순화 |
| 특정 단계에서 이탈 집중 | 특정 step 이탈 > 50% | 해당 step UI/카피 A/B 테스트 |
| 오프라인 지도 다운 저조 | < 15% | 다운로드 혜택 메시지 강화 또는 자동 다운로드 검토 |
