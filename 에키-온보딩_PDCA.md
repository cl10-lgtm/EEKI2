## P — Plan (기획 목표 및 설계 원칙)

### 기능 목표

온보딩은 에키의 핵심 가치를 **첫 60초 안에** 유저가 체감하게 만드는 유일한 기회다.
박서연처럼 사전에 준비하려는 유저와, 최민준처럼 현장에서 즉시 해결하려는 유저 모두를 놓치지 않아야 한다.
또한 경쟁사(구글맵·재팬트랜짓)와의 차별점인 **"역사 내부 + 결제 + 언어"** 를 온보딩에서 미리 경험시켜야 재방문율과 입소문으로 연결된다.

### 설계 원칙

- **선택지 최대 3개** — 정보 과다가 핵심 페인포인트이므로 온보딩도 동일 원칙 적용
- **강제 입력 없음** — 모든 설정은 "나중에 하기" 탈출구 제공, 설정 미완료 상태에서도 경로 검색 가능
- **두 가지 경로 분기** — 사전 준비형(박서연)과 즉시 실행형(최민준)을 첫 화면에서 분기
- **진행률 항상 표시** — 얼마나 남았는지 보여줘 이탈 방지

## D — Do (기능 명세)

### FEAT-01. 진입 분기 선택

**트리거**: 앱 최초 실행 완료 직후 (로그인 불필요, 즉시 노출)

**목적**: 박서연·최민준 두 유저 유형을 첫 화면에서 분기해 각자의 기대에 맞는 흐름으로 안내
> 근거 — 박서연: *"앱을 처음 켜는 순간부터 믿어도 되겠다는 신뢰감"* 필요
> 최민준: *"그냥 켜면 바로 알려주는 앱 하나 있었으면"*

**화면 구조**

```
┌─────────────────────────────────────┐
│           에키에 오신 걸 환영해요    │
│    일본 지하철, 이제 헷갈리지 마세요  │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  ✈ 여행 전에 미리 준비할게요  │    │
│  │  출발 전에 카드·역 정보 세팅  │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  📍 지금 당장 탈 곳 찾을게요  │    │
│  │  준비 없이 바로 경로 검색     │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

| 옵션 | 연결 플로우 |
|------|------------|
| 여행 전 준비 | FEAT-02 → FEAT-03 → FEAT-04 순서 진행 |
| 지금 당장 | FEAT-05 즉시 경로 검색으로 바로 이동 |

**예외 처리**
- 5초 무응답 시 옵션 B(즉시 실행)로 자동 진입 후 토스트: "나중에 설정 메뉴에서 준비할 수 있어요"
- 재실행 시 이 화면 스킵, 홈 화면 직행

**출력 결과**: 선택한 경로 값 로컬 저장 (`onboarding_path: prepare | instant`)

### FEAT-02. 교통카드 온보딩

**트리거**: FEAT-01에서 "여행 전 준비" 선택 후

**목적**: 교통카드 구매·충전·잔액 확인 정보가 파편화되어 발생하는 혼란을 사전에 제거
> 근거 — U9: *"구매처와 방법에 대한 설명 부족으로 인한 시간 요소"*
> U6: *"교통카드 잔액 표시가 안 되어서 그냥 찍었다가 잔액부족으로 뜸"*
> U5: *"결제 시 일본 앱 설치 및 애플페이 연동 혼란"*

**단계별 명세**

```
Step 1 — 카드 선택 (1/3)
┌──────────────────────────────────────┐
│  어떤 교통카드를 쓸 예정인가요?       │
│  ─────────────────────────────────  │
│  [ Suica        가장 범용적          ]│
│  [ Pasmo        도쿄·간토 지역       ]│
│  [ 트래블월렛   한국 카드로 충전      ]│
│                                      │
│  아직 모르겠어요 →  (스킵)            │
└──────────────────────────────────────┘

Step 2 — 카드별 맞춤 안내 (2/3)
  선택 카드에 따라 다른 콘텐츠 표시:
  - 구매 가능 장소 (역명 + 기기 이름)
  - 초기 충전 권장 금액
  - 사용 불가 노선 주의사항 (있는 경우)
  - 충전 ATM 찾는 법 한 줄 요약

Step 3 — 잔액 알림 설정 (3/3)
┌──────────────────────────────────────┐
│  잔액이 얼마 이하일 때 알려드릴까요?  │
│                                      │
│  [ 500엔 ]  [ 1,000엔 ]  [ 직접 입력] │
│                                      │
│  나중에 설정할게요 →                  │
└──────────────────────────────────────┘
```

**예외 처리**
- "아직 모르겠어요" 선택 시: 카드 비교 페이지 링크 제공 후 Step 3으로 바로 이동
- Step 3 스킵 시: 기본값 1,000엔으로 설정, 홈에서 변경 가능 안내

**출력 결과**: `{ card_type, alert_threshold }` 로컬 저장

### FEAT-03. 주요 역 오프라인 지도 다운로드

**트리거**: FEAT-02 완료 후

**목적**: 지하 심층부 LTE 불안정 환경에서도 역사 내부 길찾기 가능하게 함
> 근거 — U8: *"신주쿠 역같은 경우 다수의 출구로 인해 위치 찾기가 어려움"*
> U6: *"역 내가 커서 수많은 안내 위치와 포스터로 인해 헷갈림"*

**화면 구조**

```
목적지 도시 선택 → 해당 복잡 역 자동 추천

예) 도쿄 선택 →
  ☑ 신주쿠역    2.3MB  [복잡도 ●●●●●]
  ☑ 시부야역    1.8MB  [복잡도 ●●●●○]
  ☐ 도쿄역      2.1MB  [복잡도 ●●●●●]
  ☐ 우에노역    1.2MB  [복잡도 ●●●○○]

  [ 선택 역 다운로드 ]   [ 전체 다운로드 ]
  나중에 할게요 →
```

**포함 데이터 (역별)**
- 출구 번호 + 출구별 인접 시설 (편의점·화장실·코인락커·안내소)
- 환승 홈 위치 및 환승 소요 시간
- 탑승 플랫폼 번호

**예외 처리**
- 다운로드 실패(네트워크 오류) 시: "Wi-Fi 연결 후 다시 시도" 안내, 건너뛰기 가능
- 저장 공간 부족 시: 용량 경고 후 개별 선택 유도

**출력 결과**: 오프라인 역 지도 로컬 저장, 홈에서 다운로드 상태 뱃지 표시

### FEAT-04. 역무원 대화 카드 튜토리얼

**트리거**: FEAT-03 완료 후 (온보딩 마지막 단계)

**목적**: 번역 타이핑 시간과 심리적 부담을 제거, "문제가 생겨도 해결할 수 있다"는 자신감 부여
> 근거 — U6: *"오래 번역 시에 눈치 보이고, 줄에 대한 부담감이 생김"*
> U6: *"역무원과의 번역 의지 부족으로 의사소통이 안돼서 불편함"*

**튜토리얼 구조 (30초 완료 목표)**

```
슬라이드 1: "이런 상황, 겪어본 적 있으신가요?"
  → 개찰구 앞 잔액 부족 상황 일러스트

슬라이드 2: "에키 대화 카드를 써보세요"
  → 카드 실제 화면 미리보기
  ┌────────────────────────────┐
  │ 교통카드 잔액이 부족해요    │
  │ ICカードの残高が            │
  │ 不足しています              │
  └────────────────────────────┘

슬라이드 3: "이런 상황에서 바로 꺼내세요"
  상황 카드 3종 미리보기:
  • 잔액 부족으로 개찰구 막혔을 때
  • 열차를 잘못 탔을 때
  • 짐 보관함 위치를 물어볼 때
```

**카드 데이터 구조**
```json
{
  "situation_ko": "잔액 부족으로 개찰구 막혔을 때",
  "display_ko": "교통카드 잔액이 부족해요",
  "display_ja": "ICカードの残高が不足しています",
  "furigana": "アイシーカードのざんだかがふそくしています"
}
```

**예외 처리**
- 튜토리얼 중 "이미 알아요" 탭 시 즉시 스킵, 온보딩 완료 처리
- 완료 후 홈에서 언제든 "대화 카드" 탭으로 재접근 가능

**출력 결과**: 온보딩 완료 상태 저장, 홈 화면 언락

### FEAT-05. 즉시 실행 경로 검색 (최민준 경로)

**트리거**: FEAT-01에서 "지금 당장" 선택 후

**목적**: 준비 없이 현장에서 실행한 유저가 **30초 안에** 첫 경로를 얻게 함
> 근거 — 최민준: *"그냥 켜면 바로 알려주는 앱 하나 있었으면 진짜 좋겠다"*
> U9: *"구글 맵에 의존하면서도 실제 진행 방향이 맞는지 확인할 수 없어 막막함"*

**화면 구조**

```
┌──────────────────────────────────────┐
│  지금 어디서 어디로?                  │
│                                      │
│  현재 위치  [ 하카타역 (자동감지) ▼ ] │
│  목적지     [ __________________ ]   │
│                                      │
│  [ 경로 찾기 ]                        │
└──────────────────────────────────────┘

결과 화면:
┌──────────────────────────────────────┐
│  하카타 → 텐진  약 8분               │
│                                      │
│  🚇 공항선 탑승 → 2정거장            │
│  📍 탑승: 1번 플랫폼                 │
│  🚪 하차 후: 7번 출구으로 나가세요   │
│                                      │
│  ─────────────────────────────────  │
│  💳 교통카드 설정도 해볼까요?  [닫기] │
└──────────────────────────────────────┘
```

**예외 처리**
- GPS 자동 감지 실패 시: 수동 입력 모드로 전환, "현재 역 이름을 입력해주세요"
- 오프라인 상태 시: 다운로드된 역 데이터로 부분 안내, 경로 계산 불가 명시

**출력 결과**: 경로 결과 + 탑승 플랫폼 + 하차 출구 표시

### FEAT-06. 온보딩 완료 상태 관리

**목적**: 중간 이탈 유저도 다시 돌아와 완료할 수 있게 함

**명세**
- 각 FEAT 완료 여부 로컬 저장 (FEAT-01 ~ FEAT-04)
- 재진입 시 완료 단계 스킵, 미완료 단계부터 재개
- 홈 화면 상단 배너: "설정 완료하면 더 편해요" → 클릭 시 미완료 단계로 이동 → 온보딩 완전 완료 시 자동 소멸
- 설정 메뉴에서 언제든 재설정 가능

## C — Check (검증 지표)

### 정량 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| 온보딩 완료율 (P1 경로) | ≥ 60% | FEAT-01 진입 → FEAT-04 완료 funnel |
| 즉시 실행 선택 비율 | 측정 후 분기 최적화 기준으로 활용 | FEAT-01 옵션 B 선택 수 / 전체 |
| 교통카드 설정 완료율 | ≥ 45% | FEAT-02 Step 3 완료 수 |
| 오프라인 지도 다운로드율 | ≥ 30% | FEAT-03 다운로드 완료 이벤트 |
| 첫 경로 검색까지 소요 시간 | ≤ 60초 | 앱 실행 타임스탬프 → 첫 결과 화면 타임스탬프 |
| 온보딩 이탈 후 복귀율 | ≥ 25% | FEAT-06 홈 배너 클릭 → 완료 전환 |

### 정성 지표 (출시 후 2주 내 인터뷰)

- "이 앱 켰을 때 뭔가 다르다는 느낌을 받았나요? 어느 순간이었나요?"
- "온보딩 중 멈추거나 헷갈린 지점이 있었나요?"
- "역무원 대화 카드를 실제로 꺼내 쓴 적 있나요? 효과가 있었나요?"

## A — Act (개선 트리거 및 로드맵)

### 즉시 개선 트리거

| 상황 | 기준 | 액션 |
|------|------|------|
| P1 경로 온보딩 완료율 저조 | < 40% | FEAT-02 Step 수 3→2 축소 검토, 카피 A/B 테스트 |
| 즉시 실행 선택 압도적 | > 80% | P1 사전 준비 경로를 설정 메뉴로 이동, 온보딩 단순화 |
| 특정 Step 이탈 집중 | 특정 step 이탈 > 50% | 해당 step UI·카피 A/B 테스트 우선 진행 |
| 오프라인 지도 다운 저조 | < 15% | 다운로드 혜택 메시지 강화 또는 자동 다운로드 검토 |
| 대화 카드 실사용률 낮음 | < 10% | 튜토리얼 진입 시점·시나리오 재검토 |

```
MVP (v1.0)  ── 가장 빠르게, 가장 중요한 것만

  └── FEAT-02 교통카드 온보딩 Step 1~2만

v1.1  ── 데이터 보고 판단
  ├── FEAT-03 오프라인 지도 다운로드

  └── FEAT-02 Step 3 잔액 알림 설정 추가

v1.2  ── 감정적 차별화 완성
  └── FEAT-04 역무원 대화 카드 튜토리얼
```
